1.Linear Search

Linear search scans each element in a list sequentially until the target element is found or the end of the list is reached.

Algorithm:

1.Start at the beginning of the array/list: Set an index i = 0, which will point to the first element in the list.
2.Check if the current element is equal to the target value: Compare the element at position i with the target value you're searching for.
3.If a match is found: If the element at position i matches the target value, stop the search and return the index i.
4.If no match is found: Move to the next element by increasing the index i by 1 (i = i + 1). Repeat steps 2 and 3 for the next element.
5.Check if the entire array has been searched: If you reach the end of the list (i.e., i is equal to the length of the list), and no match has been found, conclude that the target value is not present in the list.
6.Return the result: If the target value was found in the list, return its index. If the target value was not found, return an indication that the value is not present, such as -1 (or any suitable value that represents "not found").

Time Complexity

Case Time Complexity
Best Case O(1)
Average Case O(n)
Worst Case O(n)

2.Binary Search

Binary search works on sorted arrays by repeatedly dividing the search interval in half until the target element is found.

Algorithm

1.Initial Setup: Ensure the array or list is sorted. Define two pointers: low (starting at the first element of the array) and high (starting at the last element of the array).

Find the middle element: Calculate the middle index using the formula: mid=(low+high)/2 In an array, the middle index would be: mid= low+(high-low)/2
3.Compare the target value with the middle element: If the middle element is equal to the target, return the middle index. The search is complete. If the target is smaller than the middle element, eliminate the right half of the array by updating high = mid - 1. If the target is larger than the middle element, eliminate the left half of the array by updating low = mid + 1.
4.Repeat the process: Repeat steps 2 and 3 with the updated low and high values until the target is found, or the search space is empty (low > high).
5.Return the result: If the target is found, return its index. If low exceeds high (meaning the target is not found), return an indication that the value is not present (e.g., return -1).

Time Complexity

Case Time Complexity
Best Case O(1)
Average Case O(log n)
Worst Case O(log n)
