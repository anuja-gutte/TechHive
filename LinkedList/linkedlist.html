Linked Lists

What is a Linked List?

A Linked List is a linear data structure where each element, called a node, is an object that contains data and a reference to the next node. Unlike arrays, linked lists do not store data in contiguous memory locations. This allows for more efficient insertion and deletion operations compared to arrays.

In a linked list, elements are connected using pointers, and each node contains two main components:

  • Data: The value or data stored in the node.
  • Pointer (or link): A reference to the next node in the sequence.

Advantages of Linked Lists

Linked lists have several advantages over arrays:

  • Dynamic Size: Linked lists can grow or shrink during runtime as needed, while arrays have a fixed size.
  • Efficient Insertions/Deletions: Inserting or deleting nodes in a linked list is more efficient, as it only involves updating pointers.
  • Memory Utilization: Memory is allocated as needed, so there is no wasted space as with arrays where you may need to predefine the size.

Types of Linked Lists

There are several types of linked lists, each with its unique structure and application:

  • Singly Linked List: In this type of linked list, each node has data and a reference to the next node in the sequence. It allows traversal in one direction, from the head (first node) to the tail (last node).
  • Doubly Linked List: Each node contains references to both the next and the previous nodes. This allows traversal in both directions (forward and backward).
  • Circular Linked List: In a circular linked list, the last node in the sequence points back to the first node, creating a circular structure.

Basic Structure of a Node

The basic structure of a linked list node can vary based on the type of linked list. Below is an example of a node structure for a singly linked list in C:

struct Node {
    int data;
    struct Node* next;
};

In this structure, data holds the value stored in the node, and next is a pointer to the next node in the list.

How Linked Lists Work

In a linked list, the first node is referred to as the head, and the last node is referred to as the tail. The tail node's pointer typically points to NULL, indicating the end of the list (except in circular linked lists).

  • Deletion: To delete a node, we modify the pointers of the surrounding nodes to bypass the node being deleted.
  • Traversal: We can traverse a linked list by following the pointers from the head node to the tail.

Common Operations in Linked Lists

Some common operations performed on linked lists include:

  • Insertion: Inserting an element at the beginning, middle, or end of the list.
  • Deletion: Deleting an element from the list.
  • Traversal: Visiting each element in the list to perform a specific task (e.g., searching for an element).
  • Reversal: Reversing the order of nodes in the list.

Applications of Linked Lists

Linked lists are widely used in various applications due to their flexibility and efficiency. Some common applications include:

  • Dynamic Memory Allocation: Linked lists are used to manage free memory in operating systems.
  • Implementing Stacks and Queues: Linked lists provide a convenient way to implement stacks and queues dynamically.
  • Graph Representations: In graph data structures, adjacency lists are often implemented using linked lists.
  • Polynomial Manipulation: Linked lists can be used to store and manipulate polynomials where each node represents a term of the polynomial.
